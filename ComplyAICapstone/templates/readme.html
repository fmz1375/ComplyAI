About ComplyAI

Documentation & Reference Guide

ComplyAI

A NIST CSF Cybersecurity Compliance Assessment Platform with AI-Powered Analysis

A comprehensive Flask-based web application for automated NIST Cybersecurity Framework (CSF) compliance analysis, document management, and risk assessment using Advanced RAG (Retrieval-Augmented Generation) technology.

Quick Start Guide (For Professors / Evaluators)

Prerequisites

  • • Python 3.10+ installed
  • • Ollama installed and running (https://ollama.com)
  • • Git (for cloning the repository)

Installation Steps

1

Clone the repository

git clone https://github.com/YOUR_USERNAME/complyai.git
cd complyai
2

Create and activate a virtual environment

# Windows
python -m venv .venv
.venv\Scripts\activate

# macOS/Linux
python3 -m venv .venv
source .venv/bin/activate
3

Install Python dependencies

pip install -r requirements.txt
4

Pull required Ollama models

Run these commands in a separate terminal:

ollama pull qwen2.5:3b
ollama pull saki007ster/CybersecurityRiskAnalyst
5

Start the application

python appOne.py
6

Open your browser and navigate to:

http://127.0.0.1:5005

Key Features to Test

1 Upload a Policy Document
  • • Navigate to "Upload Document" from the sidebar
  • • Select a PDF security policy (e.g., Information Security Policy)
  • • Fill in metadata: name, category, priority, due date
  • • System extracts text and prepares for AI analysis
2 Run Compliance Analysis
  • • Click "Analyze" to start AI-powered gap detection
  • • Watch real-time progress as the system processes
  • • Uses Advanced RAG with FAISS + BM25 hybrid retrieval
  • • Review gaps with severity ratings (Critical, High, Medium, Low)
  • • Provide feedback to improve future accuracy
3 Complete NIST CSF Questionnaire
  • • Choose NIST CSF functions: Govern, Identify, Protect, Detect, Respond, Recover
  • • Answer structured questions about your security posture
  • • Multiple question types: multiple choice, yes/no, scale, text
  • • Progress is saved automatically
4 Generate Reports
  • • Choose export format: PDF or Word (DOCX)
  • • Includes: executive summary, compliance gaps, risk assessment
  • • Edit report content before finalizing
  • • Share with team or download for offline use
5 Risk Heat Map
  • • Complete Risk Appetite questionnaire for tolerance levels
  • • View 5x5 matrix: likelihood vs. impact for each risk
  • • Items exceeding risk appetite are highlighted
  • • Download heat map as PNG for presentations
6 Real-time Collaboration
  • • Share documents via email invitation or shareable link
  • • Set permissions: view-only, comment, or full edit
  • • Real-time chat within document assessment rooms
  • • Direct messaging (DM) for private conversations
  • • See who's online and typing indicators

Security Features

ComplyAI implements comprehensive security hardening to protect against common attack vectors:

Attack Type Protection Method Implementation
SQL Injection Parameterized queries + table whitelist security_utils.SQLInjectionDefense
Prompt Injection Input sanitization + pattern detection security_utils.PromptInjectionDefense
Buffer Overflow Input length limits (5000 chars max) security_utils.InputValidation
SSRF URL validation + private IP blocking InputValidation.validate_url()
XSS Jinja2 auto-escaping Flask templates

Protected Components

  • LLM Service - All inputs sanitized
  • Context Builder - Pattern detection
  • RAG Chatbot - Query sanitization
  • Database - Parameterized queries

Password Security

  • Werkzeug password hashing
  • Secure Flask sessions
  • Optional Auth0 SSO integration

Troubleshooting

"Ollama not available" warning

Make sure Ollama is running: ollama serve

Port already in use

Change the port in appOne.py or kill the existing process

Missing dependencies

Run pip install -r requirements.txt again

FAISS index not found

Run python nist_embedding.py to rebuild the index

Database errors

Delete project.db and restart the application to recreate
