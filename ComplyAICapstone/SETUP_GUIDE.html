Setup & Installation Guide

Version 1.0 | April 2026

1. Prerequisites

Before installing ComplyAI, ensure you have the following software installed on your system:

Software Version Download Link
Python 3.10 or higher https://www.python.org/downloads/
Ollama Latest https://ollama.com/download
Git Latest https://git-scm.com/downloads
ℹ️ Note
Ollama is required for AI-powered compliance analysis. Make sure it's running before starting ComplyAI.

2. Installation Steps

1 Clone the Repository

Open a terminal (Command Prompt, PowerShell, or Terminal) and run:

git clone https://github.com/YOUR_USERNAME/ComplyAI.git
cd ComplyAI
2 Create a Virtual Environment

Windows:

python -m venv .venv
.venv\Scripts\activate

macOS / Linux:

python3 -m venv .venv
source .venv/bin/activate
ℹ️ Tip
You should see (.venv) at the beginning of your terminal prompt when the virtual environment is active.
3 Install Python Dependencies
pip install -r requirements.txt

This will install Flask, SQLAlchemy, FAISS, and all other required packages.

4 Pull Required AI Models

Open a new terminal window and run these commands:

ollama pull qwen2.5:3b
ollama pull saki007ster/CybersecurityRiskAnalyst
⚠️ Important
The model downloads may take several minutes depending on your internet speed. Each model is approximately 2-4 GB.
5 Start the Application
python appOne.py

The server will start on port 5005.

6 Access the Application

Open your web browser and navigate to:

http://127.0.0.1:5005
✅ Success! You should see the ComplyAI login page.

4. Key Features to Test

4.1 Upload a Policy Document

  1. Navigate to "Upload Document" from the sidebar
  2. Select a PDF security policy document
  3. Fill in metadata: document name, category, priority, and due date
  4. The system extracts text and prepares for AI analysis

4.2 Run Compliance Analysis

  1. After upload, click "Analyze" to start AI-powered gap detection
  2. Watch real-time progress as the system processes your document
  3. Uses Advanced RAG with FAISS + BM25 hybrid retrieval
  4. Review identified gaps with severity ratings (Critical, High, Medium, Low)

4.3 Complete NIST CSF Questionnaire

  1. Access from the document analysis page or dashboard
  2. Choose NIST CSF functions: Govern, Identify, Protect, Detect, Respond, Recover
  3. Answer structured questions about your security posture
  4. Progress is saved automatically

4.4 Generate Reports

  1. After completing analysis, generate comprehensive reports
  2. Choose export format: PDF or Word (DOCX)
  3. Reports include: executive summary, compliance gaps, risk assessment, recommendations
  4. Edit report content before finalizing

4.5 Risk Heat Map

  1. Access from the document viewer or report page
  2. Complete the Risk Appetite questionnaire
  3. View 5×5 matrix showing likelihood vs. impact
  4. Download heat map as PNG for presentations

5. Troubleshooting

Problem Solution
"Ollama not available" warning Make sure Ollama is running. Open a terminal and run: ollama serve
Port 5005 already in use Kill existing process or change port in appOne.py
Missing dependencies error Run pip install -r requirements.txt again
Model not found error Run ollama pull qwen2.5:3b to download the model
Database errors Delete project.db and restart the app to recreate

6. Project Structure

├── appOne.py              # Main Flask application (entry point)
├── requirements.txt       # Python dependencies

│   └── risk_heatmap_service.py  # Risk visualization
├── rag/                  # RAG (Retrieval-Augmented Generation)
│   └── advanced_rag.py          # Hybrid retrieval + gap analysis
├── models/               # Pydantic data models
├── faiss_index/          # Vector store indexes
├── uploads/              # Uploaded documents
└── exports/              # Generated reports
